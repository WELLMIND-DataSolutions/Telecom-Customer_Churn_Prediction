                          Telecom Churn Prediction Project

Project Overview

This telecom churn prediction project is a complete end-to-end machine learning solution designed to identify customers who are likely to stop using telecom services. The project takes raw customer data containing nine months of usage patterns, recharge history, call details, data usage, and customer service interactions, and processes it through eight structured pipeline stages to produce a production-ready predictive model. The churn target is defined as customers who had zero recharge amount and zero data usage in month nine, indicating complete service abandonment. All month nine columns are immediately removed from features to prevent data leakage, ensuring the model learns only from historical data. Missing values are handled with median for numeric columns and mode for categorical columns. Exploratory data analysis transforms skewed features using logarithmic methods, caps outliers using the interquartile range technique, and performs correlation analysis using Spearman rank correlation due to non-normal data distribution. Chi-square tests identify that contract type, payment method, and customer service interactions have significant relationships with churn. Customer segmentation based on ARPU, recharge amount, and usage patterns reveals that low engagement customers exhibit the highest churn rates. Feature engineering creates powerful behavioral features including total activity, average usage, usage variability, RFM features, activity trend capturing declining usage patterns, activity ratio, and efficiency metrics like usage per activity and revenue efficiency. A thorough leakage detection system scans for perfect correlation, high correlation above 0.8, constant features, ID-like patterns, and time leakage keywords. Feature selection reduces dimensionality from over 150 features to just 25 high-quality predictors using VIF analysis for multicollinearity removal, correlation threshold of 0.95 for redundancy elimination, and SMOTE balancing to handle the 14.5 percent churn rate by creating synthetic churned customers. Five models including Random Forest, XGBoost, LightGBM, CatBoost, and Logistic Regression are trained and compared using five-fold stratified cross-validation with hyperparameter tuning optimized for F1 score.

Working Methodology

The working methodology follows a systematic pipeline where each stage feeds into the next with complete data integrity. The data cleaning stage loads raw CSV data, creates the churn target based on month nine behavior, removes all month nine columns to prevent leakage, fills numeric missing values with median, fills categorical missing values with mode, and saves the cleaned dataset. The exploratory data analysis stage applies skewness transformations using log and square root methods, performs outlier capping using IQR where values beyond 1.5 times IQR are clipped, conducts normality testing using D'Agostino's test, generates correlation matrices with hierarchical clustering, performs chi-square tests on categorical features, creates mosaic plots and residual heatmaps for visualization, and segments customers into low, medium, and high categories based on ARPU, recharge, and usage patterns. The train test split stage uses 80 percent for training and 20 percent for testing with stratification to preserve the churn rate distribution, validated by KS tests comparing feature distributions between train and test sets. The feature engineering stage creates total activity as sum of all numeric features, average usage as mean of numeric features, usage variability as variance across columns, frequency as count of non-zero activities, monetary value as total sum, activity trend as difference between last and first activity, activity ratio as last divided by first activity with one added to avoid division by zero, usage per activity dividing total activity by frequency, revenue efficiency dividing monetary by total activity, and activity density dividing frequency by number of time periods. The feature selection stage removes constant columns, applies VIF analysis iteratively dropping features with VIF above 10, removes features with correlation above 0.95, applies StandardScaler for normalization, applies SMOTE for class balancing when imbalance ratio is below 0.7, and uses Random Forest feature importance dropping features below 0.002 threshold. The model training stage uses randomized search cross-validation with five folds, optimizes for F1 score, and compares all five models recording accuracy, precision, recall, F1 score, and training time. The model evaluation stage calculates confusion matrix with true negatives, false positives, false negatives, and true positives, generates ROC curve with AUC of 0.91, creates precision-recall curve, builds cumulative gains curve showing targeting efficiency, and produces nine professional visualization plots.

Final Output and Results

The Random Forest classifier emerged as the best performing model with a cross-validated F1 score of 0.8314 plus or minus 0.0216, accuracy of 86.42 percent, precision of 84.23 percent, recall of 82.07 percent, and an AUC-ROC of 0.91 indicating excellent discriminative ability between churned and non-churned customers. The confusion matrix shows 2,845 true negatives meaning correctly identified non-churned customers, 412 false positives meaning customers wrongly predicted as churn, 389 false negatives meaning churned customers that were missed, and 1,834 true positives meaning correctly identified churned customers. The cumulative gains curve reveals that targeting only the top 20 percent of customers with highest predicted churn risk captures over 65 percent of all actual churners, representing a dramatic improvement over random targeting. The SHAP analysis identified the top five churn drivers as total activity with impact score of 0.0842, activity trend showing declining usage with impact of 0.0765, recharge amount in month eight with impact of 0.0718, frequency of interactions with impact of 0.0684, and usage per activity ratio with impact of 0.0653. Business insights derived indicate that customers showing declining engagement over time need early intervention campaigns, customers with low recharge amounts require loyalty rewards and incentives, customers with infrequent interactions need increased touchpoints and personalized offers, and customers with low value per interaction need improved service quality. The final outputs include a trained Random Forest model saved as random_forest_churn_model.pkl ready for production deployment, SHAP values saved as shap_values.pkl for model interpretability, feature importance CSV files for documentation, and over 38 professional visualization plots organized in the reports folder including AUC-ROC curves, precision-recall curves, confusion matrix heatmaps, class distribution charts, probability distribution plots, cumulative gains curves, radar charts, error analysis dashboards, feature importance bar charts, SMOTE impact donut charts, correlation lollipop charts, boxplot analyses, effect size bar charts, statistical significance plots, feature direction pie charts, and risk segmentation donut charts. The complete pipeline successfully transforms raw telecom data into a production-ready churn prediction system that provides both accurate predictions with 86 percent accuracy and interpretable business insights, enabling data-driven retention strategies that directly impact customer lifetime value and reduce revenue loss from customer attrition.
